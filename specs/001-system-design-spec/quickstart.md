# Quickstart: 多地交付数据工厂系统规格落地

## 目标
在一个迭代周期内完成“规格对齐 -> 子仓映射 -> 门禁生效 -> 首轮交付”闭环。

## 步骤
1. 核心仓发布系统规格
- 评审并冻结 `/specs/001-system-design-spec/spec.md`
- 在核心仓公告双业务基座能力域清单与边界规则

2. 五个子仓完成映射
- 每个子仓提交“能力域映射表 + 本地差异说明”
- 明确哪些需求属于平台复用，哪些属于地区定制

3. 建立统一评审门
- 在各仓启用统一评审模板（业务/技术/合规/发布）
- 指定客户方与丰图方角色责任

4. 启动首轮交付包
- 每个地区选择一个最小交付包作为试点
- 按统一门禁推进并记录全过程

5. 复盘与迭代
- 汇总 KPI：交付周期、复用率、返工率、自动化覆盖率
- 对边界冲突与流程卡点给出调整方案

## 完成标准
- 5 个子仓均提交映射并通过一次联评
- 至少 1 个跨仓可复用能力完成闭环交付
- 关键评审记录可完整追溯
