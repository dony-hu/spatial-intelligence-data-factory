# 旧目录结构迁移说明

> 文档状态：历史说明
> 作用：记录 `docs/` 从旧英文目录结构迁移到新编号文档体系的规则

## 1. 本次迁移结论

以下目录已退役，不再作为正式文档入口：

1. `docs/product/`
2. `docs/architecture/`
3. `docs/测试/`
4. `docs/project-management/`

它们原先承载的正式文档，现已归口到新的编号目录。

1. 总入口：`docs/00_阅读指南.md`

11. 项目管理：`docs/12_项目管理/`

## 3. Epics 目录的例外规则

`docs/研发过程管理/` 保留为独立阶段文档区，不直接迁移或并入阅读型目录。

原因：

1. 它承载按研发主题分目录的阶段性交付材料。
2. 它更接近执行管理区，而不是稳定的阅读型正式文档。
3. 直接合并会破坏阶段性文档的组织边界。

## 4. 使用规则

1. 新正式文档应优先进入编号目录。
2. 新阶段性文档应进入 `docs/研发过程管理/`。
3. 不得继续向旧英文目录结构新增正式文档。
