# Brainstorm：空间智能数据工厂（2026-02-28）

> 文档状态：历史归档
> 归档原因：头脑风暴过程记录，已由当前 `product-brief`、`PRD`、`架构真相源` 吸收有效结论
> 归档日期：2026-03-06

## 1. 目标
围绕“地址治理闭环 + PG-only 收敛 + 可观测运营”三条主线，识别可在 1-2 个迭代内落地、且能显著降低交付风险的产品化动作。

## 2. 头脑风暴候选方向

### 方向 A：治理任务 SLA 与异常分级联动
- 核心想法：给每类治理任务定义 SLA（如 T+5 分钟完成），超时自动分级告警并记录责任边界。
- 价值：把“任务完成”升级为“按时完成”，减少业务侧不确定性。
- 代价：需要在执行链路补充时钟/分级规则与告警通道。

### 方向 B：结果证据标准化卡片（Evidence Card）
- 核心想法：对 `strategy/confidence/evidence` 统一渲染为可检视证据卡，支持抽样复核。
- 价值：降低人工审核与审计沟通成本。
- 代价：需要统一字段字典与样式模板。

### 方向 C：规则变更沙箱与回放（Shadow Replay）
- 核心想法：规则上线前先对历史样本回放，输出“命中率变化/误伤率变化”。
- 价值：降低规则迭代引发的生产波动。
- 代价：需要样本集维护和回放资源。

### 方向 D：多租户/多地区规则包版本治理
- 核心想法：统一规则包版本与发布窗口，支持灰度和回滚。
- 价值：控制跨地区配置漂移，减少配置事故。
- 代价：需要引入版本依赖校验与发布流程约束。

### 方向 E：运营看板增加“治理漏斗”视图
- 核心想法：按“输入 -> 解析成功 -> 规则命中 -> 人工复核 -> 最终通过”展示漏斗。
- 价值：帮助业务快速定位瓶颈点。
- 代价：需要补齐分层指标定义和数据口径。

### 方向 F：CI 增加“契约漂移拦截”硬门
- 核心想法：对关键输出 JSON schema 做版本对比，发现破坏性变更直接阻断。
- 价值：提前发现接口兼容性风险。
- 代价：需维护 schema 基线与兼容策略。

## 3. 评估矩阵（Impact vs Effort）

高影响/低到中成本（优先）：
1. 方向 F（契约漂移拦截）
2. 方向 E（治理漏斗看板）
3. 方向 A（SLA 分级联动）

高影响/中高成本（次优先）：
1. 方向 C（规则回放）
2. 方向 D（多地区版本治理）

中影响/低成本（快速增益）：
1. 方向 B（证据标准化卡片）
