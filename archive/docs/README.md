> 文档状态：历史归档入口

## 1. 目的

本目录用于承接已退出现行文档体系、但仍需保留追溯价值的历史文档。

## 2. 当前分区

1. `archive/docs/architecture/`
   - 历史架构基线与被降级的旧架构文档。
2. `archive/docs/formal/`
   - 被 canonical 文件替代的正式文档旧版本。
3. `archive/docs/研发过程管理/`
   - 研发主题目录下已被新正式文件吸收的 dated 评审、UX、Acceptance 等旧版本。
4. `archive/docs/discovery/`
   - 头脑风暴、探索性资料。
5. `archive/docs/reviews/`
   - 一次性评审、确认、进展、对齐报告。
6. `archive/docs/`
   - 更早期历史资料，保持原样保留。

## 3. 使用规则

1. 归档文档默认不作为当前需求、架构、Story、验收或实现的直接依据。
2. 如需引用归档内容，必须明确标注“历史参考”。
3. 当前有效口径以 `docs/README.md` 和现行 canonical 文件为准。
