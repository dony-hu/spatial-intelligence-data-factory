# 执行提示词（发给项目管理看板工作线）

你负责执行工作包：`wp-pm-dashboard-test-progress-v0.1.0`。

## 目标

在看板首页底部新增第 5 区块“整体测试进展”，让管理层可直接看到测试总体状态，并一键进入测试详细面板。

## 必须满足的交付标准

1. 底部新增第 5 区块并上线。
2. 展示指标：总/执行/通过/失败/跳过/通过率/最近执行/模式。
3. 详情链接可用：
   - `http://127.0.0.1:8000/v1/governance/lab/coverage/view`
   - `http://127.0.0.1:8000/v1/governance/lab/coverage/data`
4. 缺字段不报错，显示 `-`。
5. 提供变更说明 + 截图 + 自测记录。

## 排期

- 开发：0.5 天
- 联调与验收：0.5 天
- 总计：1 天内完成

## 风险控制

1. 如果数据文件未同步新增字段，必须用兜底显示，不得抛异常。
2. 新渲染函数异常不得阻塞看板其余模块刷新。

## 回传格式（必须）

1. Done
2. Next
3. Blocker
4. ETA（绝对时间）
5. Test Report（命令 + 结果）
6. Evidence（变更说明路径、截图路径、自测记录路径）
7. Release Decision（GO/NO_GO）
