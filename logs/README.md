## 📊 周度知识提炼

### 最重要的 4 个认知

1. **AI Coding ≠ 智能系统**（周三确认）
   - 用 LLM 写代码 ✓ 有效
   - 构建自主决策系统 ✗ 需要真正的 Agent 架构

2. **Mock 数据的局限**（周五暴露）
   - 框架代码可生成
   - 业务逻辑需要真实数据验证

3. **不要重造基础设施**（周六决策）
   - 自举型 Agent 复杂度无限
   - 应用成熟的 LLM 服务而非自研

4. **工程管理比技术创新更关键**（周日体悟）
   - 好想法 + 差执行 = 失败
   - 清晰的流程 & 角色 = 加速迭代

### 周度最佳实践

- ✅ 每日反思与问题记录（有助于及时调整）
- ✅ 在问题暴露时快速决策切换（避免浪费时间）
- ✅ 先构建管理框架再扩大规模（基础很重要）
- ✅ 让真实场景驱动设计改进（Mock 容易自欺）

## 📝 日志管理规范

### 日志格式
每个日志文件遵循以下结构：
```yaml
---
log_type: project_daily | project_weekly
date: YYYY-MM-DD
timezone: Asia/Shanghai
author: 贡献者 ID
role: 岗位角色
updated_at: ISO Timestamp
---
```

### 内容模块
- 工作主题（Day）/ 周度主题（Week）
- 工作项（Task ID + Title + Status）
- 关键交付物
- 风险 & 挑战
- 后续行动（Owner + Timeline）
- 反思与认知更新

### 更新频率
- **日志**：每日 EOD 更新（18:00-23:00）
- **周报**：每周日 24:00 前完成（包括下周预期）
