## 1. 什么是“工作包蓝图”

工作包蓝图是符合 `workpackage_schema.v1` 的结构化契约，回答“系统要构建什么”，不是最终代码，也不是运行结果。

## 2. 为什么要分 Factory Agent 状态机和 Runtime 状态机

前者解决目标对齐、门禁和人工介入；后者解决任务执行、证据和运行推进。两者职责不同，不能混成一套状态。

## 3. 为什么数据库要分多个 schema

因为治理业务结果、发布状态、控制态、可信数据和审计证据属于不同职责域。分域是为了避免跨界，不是为了表面好看。

## 4. 为什么不能直接查 `trust_db.*`

`trust_db` 是过渡底座，不是现行正式入口。新实现应以 `trust_data.*` 为正式查询域。

## 5. 为什么不能把 `control_plane.*` 当业务结果表

`control_plane` 保存的是执行控制态和证据态，不是治理业务结果态。把它当主结果表会把控制语义和业务语义混淆。

## 6. 为什么文档里的流程图默认纵排

因为大多数正式设计文档在页面阅读场景下更适合纵向浏览，横排图更容易超出页面宽度并降低可读性。
