# 架构引用迁移说明

## 1. 本次变更目标

统一架构文档引用入口，停止将增量架构稿和历史架构文档当作现行基线使用，并把稳定结论归口到正式章节。

## 2. 新的引用入口

自 2026-03-07 起，架构引用统一从以下入口开始：

1. `docs/02_总体架构/架构索引.md`

当前有效架构真相源为：

1. `docs/02_总体架构/系统总览.md`
2. `docs/02_总体架构/数据工厂技术架构.md`
3. `docs/02_总体架构/模块边界.md`
4. `docs/02_总体架构/依赖关系.md`

## 3. 被降级为历史参考的文档

1. `archive/docs/architecture/架构增量收敛-历史参考.md`
2. `archive/docs/architecture/地址治理MVP架构刷新-历史参考.md`
3. `archive/docs/architecture/architecture-unified-pg-multi-schema-v1-2026-02-27.md`
4. `archive/docs/architecture/architecture-spatial-intelligence-data-factory-2026-02-27.md`
5. `archive/docs/architecture/architecture-spatial-intelligence-data-factory-2026-02-28.md`

降级后规则：

1. 可用于追溯背景、专题设计、迁移上下文。
2. 不可再写成“当前基线”“唯一基线”“默认依据”。
3. 若必须引用历史文档，需同时补充当前有效架构真相源引用。

## 4. 模板约束变更

Story 模板现已新增并强制要求填写：

1. 所属面
2. 允许依赖
3. 禁止依赖
4. 架构真相源

未填写这些字段的 Story，不应进入开发实现阶段。

1. 当前有效架构专题已经分散归口到 `02_总体架构/`、`03_数据处理工艺/`、`04_系统组件设计/`、`06_系统运行与运维/`、`09_附录/`。
