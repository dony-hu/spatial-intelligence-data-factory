## 2. 总体架构图

图说明：这张图是一页式全景图，适合新成员先看“用户发起、Agent 编排、Runtime 执行、证据沉淀”四段主链如何连起来。
```mermaid
flowchart TB
  subgraph CP["控制面"]
    CLI["Factory CLI"]
    API["Governance API / Observability API"]
    AGENT["Factory Agent"]
  end

  subgraph EP["执行面"]
    ORCH["Runtime Orchestrator"]
    WORKER["Governance Worker"]
    EXEC["Workpackage Executor"]
  end

  subgraph AP["资产面"]
    SCHEMA["workpackage_schema"]
    BUNDLE["workpackages/bundles"]
    TRUST["Trust Hub"]
    CORE["address_core\n过渡共享原语"]
  end

  subgraph XP["证据面"]
    PG["PostgreSQL"]
    REDIS["Redis / RQ"]
    OBS["Observability"]
    OUT["Output / Dashboard"]
  end

  USER["用户"] --> CLI
  USER --> API
  CLI --> AGENT
  API --> AGENT
  AGENT --> ORCH
  ORCH --> WORKER
  WORKER --> EXEC
  EXEC --> BUNDLE
  BUNDLE -. 遵循 .-> SCHEMA
  EXEC -. 查询 .-> TRUST
  BUNDLE -. 复用 .-> CORE
  AGENT --> PG
  WORKER --> PG
  ORCH --> REDIS
  WORKER --> REDIS
  PG --> OBS
  OBS --> OUT
```

## 3. 如何读这张图

1. 从左上到右下看，是“用户发起 -> 编排 -> 执行 -> 留痕”。
2. 四个面共同组成系统，不应把某一个模块当成整个系统。
3. Worker 不直接绑定治理算法，而是执行工作包入口。
