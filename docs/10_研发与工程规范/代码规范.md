# 代码规范

> 文档状态：当前有效
> 角色：工程实现与评审正式规范
> 关联文档：
> - `AGENTS.md`
> - `docs/02_总体架构/模块边界.md`
> - `docs/05_数据模型设计/数据库跨界约束.md`

## 1. 适用范围

本规范用于指导主仓代码、脚本、迁移、测试和文档协作。

## 2. 基本原则

1. 先测试，再实现。
2. 模块边界优先于局部便利。
3. 不用 fallback 掩盖真实失败。
4. 正式真相源优先于临时产物。

## 3. 分层规范

1. CLI 不直连数据库。
2. 页面不直连数据库。
3. Agent 不直写治理结果表。
4. Worker 不直接绑定治理算法模块。
5. Repository/DAO 收敛数据访问。

## 4. 数据访问规范

1. 新查询优先使用正式 schema 口径，不允许新增 `trust_db.*` 直连。
2. 非迁移场景不得随意写 DDL。
3. JSON 字段用于扩展上下文，不得替代主键和核心业务字段。

## 5. 测试规范

1. 每次改动优先补失败用例或验收检查项。
2. 关键状态机、数据库边界、API 契约必须有对应测试。
3. 阻塞、审计、证据链是必测项，不是可选项。

## 6. 文档规范

1. 正式文档用中文。
2. 文件名不用日期后缀。
3. Mermaid 流程图默认纵排。
4. 图注必须说明“这张图看什么”，不能只写通用模板句。
