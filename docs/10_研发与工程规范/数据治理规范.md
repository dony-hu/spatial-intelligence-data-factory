# 数据治理规范

> 角色：治理研发约束
> 来源：`AGENTS.md`、`docs/02_总体架构/核心设计原则.md`

## 1. 核心规范

1. 真实链路优先，不用 mock/fallback 伪造成功。
2. Worker 只按工作包入口执行。
3. 新 Story 必须声明所属面、允许依赖、禁止依赖。
4. 主链路默认以 PostgreSQL 为真相源。
5. 关键动作必须留下审计和证据。

## 2. 治理研发检查表

| 检查项 | 要求 |
|---|---|
| 架构边界 | 是否说明所属面与依赖边界 |
| 工作包契约 | 是否符合 `workpackage_schema.v1` |
| 运行态 | 是否避免直接绑算法到 Worker |
| 证据 | 是否产出 audit / trace / evidence |
| 异常 | 是否显式 `blocked/error` |
