# Product Brief：空间智能数据工厂（Spatial Intelligence Data Factory）

## 1. 背景与机会
当前项目已完成 PRD、架构设计与多条实施线推进，但在“数据治理闭环可见性、跨模块一致性、PG-only 架构收敛”方面仍需要持续通过产品化目标统一执行优先级。项目机会点在于：

- 将地址治理、可信数据 Hub、产线执行与可观测体系整合为统一的数据工厂能力。
- 通过 PostgreSQL 多 schema 统一持久化与契约校验，降低历史双栈（含 SQLite）带来的运维和一致性成本。
- 用可追踪工作包与治理指标看板支撑业务侧验收，形成“规则 -> 执行 -> 回传 -> 复核”的闭环。

## 2. 问题陈述
当前痛点集中在以下方面：

- 数据治理流程跨模块协同成本高，状态流转与人工审核闭环存在断点风险。
- 历史存储路径并存导致契约落地、测试基线和 CI 门禁复杂度上升。
- 业务方对治理覆盖率、处理时效、失败原因的可观测性需要进一步提升。

## 3. 产品目标（本阶段）

- 建立可执行、可审计、可回放的数据治理主链路（以地址治理为先导场景）。
- 完成 PG-only 路线关键改造，确保仓储入口、验收脚本与主线测试口径统一。
- 形成稳定的指标产出与看板展示能力，支持运营复盘和迭代决策。

## 4. 非目标（本阶段不做）

- 不在本阶段扩展全量行业场景规则库，只覆盖核心地址治理与数据工厂主链路。
- 不引入新的多数据库混合方案，避免与 PG-only 收敛目标冲突。
- 不进行大规模 UI 平台重构，优先保障流程可用性与治理结果可信度。

## 5. 目标用户与关键参与方

- 业务运营与治理团队：关注规则命中率、治理覆盖率、工单闭环效率。
- 数据工程与平台研发团队：关注契约一致性、运行稳定性、交付效率。
- 质量与审计角色：关注可追踪性、可解释性与回归风险可控。

## 6. 方案范围

- 地址治理任务状态机稳定化与结果契约强校验。
- 人工审核闭环补齐（任务回传、状态更新、可追踪记录）。
- PG-only 改造主线：仓储与服务入口硬切、验收脚本迁移、测试与 CI 门禁加固。
- 运行指标沉淀与业务看板输出（`output/dashboard/`）。

## 7. 成功指标（建议口径）

- 主链路成功率：治理任务从创建到回传闭环成功率 >= 99%。
- 契约违规率：关键输出契约校验失败率持续下降并可定位到责任模块。
- 回归质量：主线测试通过率稳定，PG-only 相关回流问题为 0。
- 业务可见性：看板按日更新，关键治理指标可用于周/月复盘。

## 8. 里程碑（与现有工作流对齐）

1. 完成 `PGO-S1` 到 `PGO-S6` 的开发与回归门禁。
2. 完成 MVP 地址治理 A1-A6 主线能力闭环。
3. 完成 `code-review` 与验收复盘，沉淀下一阶段优化清单。

## 9. 主要风险与缓解

- 风险：历史路径残留导致 PG-only 改造不彻底。
  - 缓解：通过 CI 硬门和全仓扫描规则阻断 SQLite 回流。
- 风险：跨模块改造引发隐藏回归。
  - 缓解：以故事级回归集和契约测试作为发布前门槛。
- 风险：业务侧对指标解释口径不一致。
  - 缓解：在看板与文档统一指标定义，并保留计算来源说明。

## 10. 下一步建议

- 按 BMAD 实施阶段继续推进 `dev-story`（优先 `PGO-S1`），并在每个故事完成后执行 `code-review` 与指标复核。

## 11. Brainstorm 补充结论（2026-02-28）

关联文档：`docs/brainstorm-spatial-intelligence-data-factory-2026-02-28.md`

### 11.1 优先方向（Impact/Cost 综合）

1. CI 契约漂移拦截硬门（优先级 P0）
2. 治理漏斗看板（输入 -> 解析 -> 命中 -> 复核 -> 通过）核心指标落地（优先级 P0）
3. 证据标准化卡片（strategy/confidence/evidence）与抽样复核模板（优先级 P1）

### 11.2 分阶段路线（Now / Next / Later）

- Now（本迭代）：先完成契约漂移拦截 + 漏斗指标 + 证据标准化模板。
- Next（下个迭代）：补齐 SLA 分级联动，并试点规则历史回放（Shadow Replay）。
- Later（后续阶段）：推进多地区规则包版本治理、灰度发布与回滚机制。

### 11.3 需要澄清的关键问题

- 治理任务的业务可接受 SLA 阈值（按任务类型分层）。
- 证据字段的“最小可解释集合”定义与审计口径。
- 规则回放样本集责任归属与更新频率。
- 多地区发布窗口策略（统一窗口或分区窗口）。
