# SpecKit 故事目录说明

本目录用于保存 SpecKit feature 与 BMAD 主题之间的挂钩记录。

## 1. 作用

1. 记录某个 `specs/<feature-id>/` 对应的 BMAD 主题和 Story。
2. 记录该 feature 的正式回填目标文档。
3. 记录该 feature 的验收路径和完成判定。

## 2. 命名建议

1. 使用 `SPEC-<序号>-<feature-slug>-挂钩说明.md`
2. 示例：
- `SPEC-001-workpackage-crud-search-seed-data-挂钩说明.md`
- `SPEC-002-address-normalization-pipeline-挂钩说明.md`

## 3. 最小内容

每份挂钩说明至少包含：

1. `SpecKit Feature ID`
2. `specs/` 目录路径
3. `BMAD Theme`
4. `BMAD Story ID`
5. `Promotion Targets`
6. `Acceptance Path`

本目录不替代 `specs/<feature-id>/tasks.md`，只负责建立正式流程挂钩关系。
