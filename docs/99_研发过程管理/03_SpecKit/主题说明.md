# 研发主题 - SpecKit 文档体系初始化

> 文档状态：当前有效
> 角色：SpecKit 在本项目中的 feature 级规格入口
> 生效日期：2026-03-06

## 1. 目标

在当前数据工厂文档体系下，为 SpecKit 建立一套 feature 级文档初始化与映射规则，使 SpecKit 能服务于：

1. 复杂模块的规格澄清与约束收敛。
2. 数据模型、接口契约、研究结论、任务拆解的独立维护。
3. 与 BMAD 的主题级 Story 建立稳定挂钩。
4. 规划中的细分专题文档从 feature 规格升级为正式文档。

## 2. SpecKit 在本项目中的定位

SpecKit 负责“模块级和功能级”规格细化，典型产物位于：

1. `specs/<feature-id>/spec.md`
2. `specs/<feature-id>/plan.md`
3. `specs/<feature-id>/tasks.md`
4. `specs/<feature-id>/research.md`
5. `specs/<feature-id>/data-model.md`
6. `specs/<feature-id>/contracts/`

SpecKit 不承担项目级 PRD、主题拆分、冲刺节奏管理；这些仍由 BMAD 负责。

## 3. 当前真相源与初始化基线

SpecKit 在本项目中的基线来源不是空白模板，而是以下现有文档：

1. `AGENTS.md`
2. `docs/00_阅读指南.md`
3. `docs/02_总体架构/架构索引.md`

5. `docs/10_研发与工程规范/项目目录结构.md`
6. `docs/10_研发与工程规范/数据治理规范.md`

## 4. 适用场景

以下情况优先进入 SpecKit：

1. BMAD 的 Story 已明确，但还需要细化接口和约束。
2. 某个模块需要独立维护数据模型和契约。
3. 需要把研究结论、任务拆解与验收路径绑定到单个 feature。
4. 需要把规划中的专题文档从“意图”推进到“可实现规格”。

## 5. 对“规划中文档”的处理方式

SpecKit 重点支撑以下规划项的下钻与成文：

1. `docs/03_数据处理工艺/地址标准化工艺.md`
2. `docs/03_数据处理工艺/地址真实性核验工艺.md`
3. `docs/03_数据处理工艺/空间实体治理工艺.md`
4. `docs/03_数据处理工艺/多源数据融合工艺.md`
5. `docs/08_AI能力设计/检索与召回机制.md`
6. `docs/08_AI能力设计/推理与排序策略.md`
7. `docs/08_AI能力设计/模型评估机制.md`

必要时，SpecKit 的数据模型、契约和研究结论还应升级回：

1. `docs/04_系统组件设计/`
2. `docs/05_数据模型设计/`
3. `docs/07_系统运行与运维/`
4. `docs/10_研发与工程规范/`

## 6. 与 BMAD 的协作原则

1. 先由 BMAD 决定主题、Story、优先级和验收口径。
2. 再由 SpecKit 细化 feature 级规格。
3. 当 SpecKit 结论稳定后，回填正式 `docs/` 和 BMAD 主题文档。

## 7. 当前初始化输出

本主题初始化包含：

1. `主题说明.md`：说明 SpecKit 在本项目中的定位。
2. `设计说明.md`：说明 SpecKit 与 BMAD 及正式文档的协作方式。
3. `故事/目录说明.md`：定义 feature 挂钩记录的落点约束。
4. `文档映射配置.yaml`：提供机器可读的 SpecKit 文档映射配置。
