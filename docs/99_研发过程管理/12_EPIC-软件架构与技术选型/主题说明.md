# 研发主题：软件架构与技术选型

## 1. 主题目标

围绕数据工厂下一阶段的软件实现基线，完成一轮面向架构、开发和工程落地的技术选型收敛，重点回答以下问题：

1. 各个核心模块应优先使用什么编程语言。
2. Runtime、服务契约、可观测、事件分发和 Agent 编排分别应采用什么开源骨架。
3. 现有 `FactoryAgent + nanobot + opencode` 方案应保留什么、替换什么、降级什么。
4. 哪些方案适合作为正式 Ring1 设计基线，哪些只保留为受控备选。

## 2. In Scope

1. 梳理现有正式文档、代码实现和当前主链路的技术事实。
2. 对比编程语言选择，包括 Python、Go、Rust、TypeScript 的适配边界。
3. 对比开源骨架，包括 Runtime、契约、可观测、事件总线与 Agent 框架。
4. 形成一份过程分析报告，并回填一份 Ring1 正式软件架构设计文档。

## 3. Out of Scope

1. 不在本主题内直接修改 Runtime、Agent、Worker 或前端实现代码。
2. 不在本主题内直接切换队列、中间件或 LLM 供应商。
3. 不把本次评估结论自动视为已落地实现状态；实现改造需后续 Story 承接。
