# SpecKit 文档设计说明

> 文档状态：当前有效
> 角色：SpecKit feature 规格与正式文档体系的映射设计

## 1. 设计原则

1. `specs/` 保存 feature 级规格，不直接替代 `docs/` 正式主线。
2. `.specify/` 保存模板、脚本和 Constitution 容器；其内容必须来源于正式文档，而不是相反。
3. 每个 SpecKit feature 都必须挂钩到一个 BMAD 主题或 Story。
4. 稳定知识要升级回 `docs/`，过程任务留在 `specs/<feature-id>/`。

## 2. SpecKit 推荐落点

| SpecKit 产物 | 默认位置 | 正式回填目标 |
| --- | --- | --- |
| `spec.md` | `specs/<feature-id>/spec.md` | `docs/99_研发过程管理/<主题>/故事/*.md` |
| `plan.md` | `specs/<feature-id>/plan.md` | 一般不直接回填，仅在 Story 或冲刺计划中摘要引用 |
| `tasks.md` | `specs/<feature-id>/tasks.md` | `docs/99_研发过程管理/<主题>/故事/*.md`、`docs/99_研发过程管理/冲刺计划.md` |
| `research.md` | `specs/<feature-id>/research.md` | `docs/02_总体架构/`、`docs/03_数据处理工艺/`、`docs/04_系统组件设计/`、`docs/07_系统运行与运维/`、`docs/08_AI能力设计/` |
| `data-model.md` | `specs/<feature-id>/data-model.md` | `docs/05_数据模型设计/` |
| `contracts/` | `specs/<feature-id>/contracts/` | `contracts/`、`schemas/`、`docs/04_系统组件设计/02_工作包协议/` |

## 3. Constitution 初始化约束

`.specify/memory/constitution.md` 在本项目中不应手写一套孤立原则，未来回填必须优先引用：

1. `AGENTS.md`
2. `docs/02_总体架构/核心设计原则.md`
3. `docs/10_研发与工程规范/项目目录结构.md`
4. `docs/10_研发与工程规范/数据治理规范.md`
5. `docs/11_附录/术语表.md`
6. `docs/09_测试与验收/全链路测试设计.md`

## 4. Feature 与 BMAD 的绑定方式

每个 feature 目录建议在 `spec.md` 顶部写入以下元数据：

1. `BMAD Theme`
2. `BMAD Story ID`
3. `Canonical Docs`
4. `Acceptance Path`
5. `Promotion Targets`

没有 BMAD 主题或 Story 归属的 feature，不应直接进入正式研发流程。

## 5. 规划中文档的升级策略

### 5.1 由 SpecKit 主导的规划项

1. `docs/03_数据处理工艺/地址标准化工艺.md`
2. `docs/03_数据处理工艺/地址真实性核验工艺.md`
3. `docs/03_数据处理工艺/空间实体治理工艺.md`
4. `docs/03_数据处理工艺/多源数据融合工艺.md`
5. `docs/08_AI能力设计/检索与召回机制.md`
6. `docs/08_AI能力设计/推理与排序策略.md`
7. `docs/08_AI能力设计/模型评估机制.md`

### 5.2 通过 SpecKit 支撑但回填到其他编号目录的内容

1. 数据模型细化：回填 `docs/05_数据模型设计/`
2. 契约和 schema：回填 `contracts/`、`schemas/`、`docs/04_系统组件设计/02_工作包协议/`
3. 运行与运维细节：回填 `docs/07_系统运行与运维/`
4. 研发规范沉淀：回填 `docs/10_研发与工程规范/`

## 6. 与历史 specs 融合基线的关系

当前 SpecKit 相关历史资产已经由以下文件定义基线：

1. `specs/README.md`
2. `docs/99_研发过程管理/规格融合/报告.md`
3. `docs/99_研发过程管理/规格融合/状态.yaml`

新的 SpecKit feature 必须在这套基线上继续演进，不再回到“双轨规格”模式。
