# Retrospective：Runtime Observability V2（2026-03-02）

## 1. 保留项（Keep）

1. 严格执行 TDD 顺序（先失败用例，再实现，再回归）。
2. 关键接口坚持后端统一口径，避免前端私算 KPI。
3. No-Fallback 失败语义统一，避免静默降级。

## 2. 删除项（Drop）

1. Story 状态与变更日志不一致写法（`done` vs `review` 混用）。
2. 无证据文件即宣称完成的做法。

## 3. 改进项（Improve）

1. 将研发主题验收包（json+md）作为每轮收口强制产物。
2. 将回归摘要统一输出到 `output/test-reports/epic-*-regression-summary-*.md`。
3. 增加 UI E2E 环境探针，避免发布前才发现执行环境缺失。

## 4. 下一轮硬约束（Hard Constraints）

1. 每个 story `done` 前必须有：测试命令、测试结果、证据路径。
2. `sprint-status.yaml` 与 story 文件状态必须一致。
3. 发布判定只认单一结论口径：gate/dashboard/release 同步。

## 5. 责任人

1. A-QA：维护回归矩阵与验收包模板。
2. A-ARC：维护架构边界检查清单。
3. A-SM：每轮收口前执行状态一致性检查。
