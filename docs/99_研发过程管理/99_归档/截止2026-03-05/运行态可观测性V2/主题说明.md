# 研发主题：地址治理系统运行态可观测 V2（2026-02-28）

## 1. 主题目标

在地址治理 MVP 主线上完成运行态可观测建设，确保页面展示系统运行结果与治理质量，而非研发过程状态。

## 2. 分阶段目标

### 阶段 A（业务可见价值，MVP）

范围：`S2-1 ~ S2-4 + S2-14`

1. 上线运行态核心 API 与页面主视图。
2. 页面可稳定展示任务产出、风险分布、任务下钻。
3. 样例包可一键灌入，避免空白页面。
4. 页面可观测新增治理包链路：`user/test_client -> factory_cli -> factory_agent <-> llm -> governance_runtime`。

### 阶段 B（可靠性与治理增强）

范围：`S2-5 ~ S2-9`

1. 补齐 SLI/SLO、延迟新鲜度、质量漂移、容量治理能力。
2. 补齐单租户最小权限与脱敏能力，进入稳定运营态。

## 3. In Scope

1. 运行态观测聚合 API（`summary/risk-distribution/version-compare/tasks`）。
2. 页面 IA 重构（KPI、风险、版本、明细下钻）。
3. 地址治理样例包（>=60 条）与灌入执行器。
4. 页面空态引导（可一键引导灌入样例）。
5. 可靠性与治理增强能力（S2-5~S2-9）。
6. 新增治理包链路观测 API 与页面链路面板（S2-14）。

## 4. Out of Scope

1. 研发流程管理可视化（Story/研发主题/CI 看板）。
2. 多租户权限体系（本主题 仅做单租户最小 RBAC）。
3. 历史数据迁移与升级兼容（当前阶段可清库重建）。

## 5. Story Backlog（执行顺序）

1. OBS-RUNTIME-S2-1：运行态聚合 API 与指标口径统一
2. OBS-RUNTIME-S2-2：运行态可观测页面 IA 重构与交互联动
3. OBS-RUNTIME-S2-3：地址治理样例包灌入与空态引导
4. OBS-RUNTIME-S2-4：任务下钻追溯与回归验收闭环
5. OBS-RUNTIME-S2-14：新增治理包链路观测（CLI/Agent/LLM/Runtime）与验收闭环
6. OBS-RUNTIME-S2-5：SLI/SLO 与告警策略闭环
7. OBS-RUNTIME-S2-6：数据新鲜度与端到端延迟观测
8. OBS-RUNTIME-S2-7：地址治理数据质量漂移与异常检测
9. OBS-RUNTIME-S2-8：观测数据保留分区与查询性能治理
10. OBS-RUNTIME-S2-9：观测权限、脱敏与审计合规（单租户最小 RBAC）

## 7. 分层 DoD

### MVP DoD（阶段 A）

1. S2-1~S2-4 + S2-14 全部通过验收测试。
2. 页面可直接展示运行态治理成果，不依赖研发过程指标。
3. 样例包灌入后页面非空且可下钻。
4. 页面可下钻查看新增治理包完整事件时间线与 Runtime receipt。
5. 形成 MVP 验收证据（JSON + Markdown）。

### Full DoD（阶段 B）

1. S2-5~S2-9 全部通过验收测试。
2. SLO/告警、数据新鲜度、质量漂移、容量治理能力可用。
3. 单租户最小 RBAC 与脱敏能力上线并可审计。
4. 形成 Full 验收证据（JSON + Markdown）。

## 8. 上线门禁表（Must / Should / Later）

| 类别 | 条目 | 说明 |
|---|---|---|
| Must | `S2-1~S2-4 + S2-14` 完成 | 不满足则不允许上线运行态页面 |
| Must | No-Fallback 主链路通过 | 关键失败必须显式 `blocked/error` |
| Must | 运行态 API 契约测试通过 | `runtime/*` 路由与字段契约稳定 |
| Must | 新增治理包链路 API 契约测试通过 | `workpackage-pipeline/workpackage-events/llm-interactions` 字段与脱敏规则稳定 |
| Should | `S2-5~S2-7` 完成 | 可灰度上线，建议在首个稳定版本完成 |
| Should | `S2-8` 查询性能治理 | 可灰度上线，需持续跟踪慢查询指标 |
| Later | 进阶权限与多租户扩展 | 不在本主题 范围，后续单独立项 |
| Later | 测试治理专项（S2-10+） | 已移出本主题，后续独立测试治理主题 |

## 9. 风险与应对

1. 风险：页面被研发数据污染。
   - 应对：只允许 `runtime/*` 数据源进入主视图。
2. 风险：样例包分布不稳定。
   - 应对：固定种子与固定分层配比。
3. 风险：阶段 B 扩展影响上线节奏。
   - 应对：阶段 A/阶段 B 分层验收与分层发布。
