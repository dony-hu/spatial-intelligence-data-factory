# Story：OBS-RUNTIME-S2-13 UI E2E 运行态语义迁移与 Skip 治理

## 1. 目标

将 UI E2E 从“研发看板语义”迁移到“运行态观测语义”，并对 `skipUnless/importorskip/skipTest` 建立可见化治理。

## 2. 范围

1. 替换 E2E 断言文案与页面结构检查到运行态视角。
2. 对 skip 用例输出清单（原因、依赖、责任人、到期时间）。
3. CI 增加 skip 统计门禁（超过阈值失败）。

## 3. 验收标准

1. E2E 主用例覆盖运行态核心模块（KPI/风险/版本/明细）。
2. 不再以“研发过程看板”字段作为主断言。
3. 每次 CI 输出 skip 报告并可追踪整改。

## 4. 测试要求（TDD）

1. 先补迁移后的失败 E2E 用例。
2. 再改页面与测试实现。
3. 最后在 CI 中验证 skip 门禁策略生效。
