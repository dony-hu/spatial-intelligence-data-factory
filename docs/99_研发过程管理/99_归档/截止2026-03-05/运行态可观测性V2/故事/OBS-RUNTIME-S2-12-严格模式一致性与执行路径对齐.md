# Story：OBS-RUNTIME-S2-12 严格模式一致性与执行路径对齐

## 1. 目标

统一 `OPENHANDS_STRICT` 在同步/异步执行路径的行为，消除“同功能不同严格度”的隐性漂移。

## 2. 范围

1. 梳理同步与异步路径中的严格模式设置点。
2. 统一默认策略与覆盖优先级。
3. 增加严格模式一致性测试（sync vs worker）。

## 3. 验收标准

1. 同步/异步路径对同一输入产生一致的严格模式行为。
2. 不再出现同步默认降级到 `OPENHANDS_STRICT=0` 的隐式差异。
3. 严格模式变更有审计记录与配置说明。

## 4. 测试要求（TDD）

1. 先补一致性失败用例。
2. 再调整执行路径实现。
3. 最后全链路回归验证一致性。
