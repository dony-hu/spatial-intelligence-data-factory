# Story：OBS-RUNTIME-S2-5 SLI/SLO 与告警策略闭环

## 1. 目标

建立系统运行态可观测的可靠性基线，定义 SLI/SLO 与 error budget，并形成告警分级、ACK、升级闭环。

## 2. 范围

1. 定义并落库 SLI：
   - 可用性（availability）
   - 延迟（latency p95/p99）
   - 数据新鲜度（freshness）
   - 指标正确性（correctness）
2. 定义 SLO 与 error budget 计算。
3. 告警策略：
   - 分级（P1/P2/P3）
   - 去重与抑制窗口
   - ACK 与升级链路
4. 观测页面新增“可靠性状态”区块。

## 3. 验收标准

1. 每个 SLI 都可查询最新值与时间序列。
2. SLO 违约可触发对应等级告警。
3. 告警支持 ACK，且有审计记录。
4. 告警抑制窗口生效，避免重复风暴。
5. 无 fallback 行为，策略异常时显式 `blocked/error`。

## 4. 测试要求（TDD）

1. 先补失败用例：
   - SLI 计算契约测试
   - SLO 违约判定测试
   - 告警 ACK/升级流程测试
2. 再实现计算与告警逻辑。
3. 最后跑回归并输出证据。

## 5. 依赖

1. 运行态聚合 API（S2-1）已完成。
2. 审计事件写入链路可用。
