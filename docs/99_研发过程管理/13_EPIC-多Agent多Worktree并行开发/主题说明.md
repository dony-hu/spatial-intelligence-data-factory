# 研发主题：多Agent多Worktree并行开发

## 1. 主题目标

为接下来的大规模重构建立一套稳定的并行研发模式，使多个 Agent 能在多个 Git Worktree 中同时工作，而不会互相污染代码、环境、契约和正式文档。

本主题解决的核心问题是：

1. 当前仓库已经进入高耦合重构阶段，单工作区串行推进会严重拖慢节奏。
2. 多 Agent 并行开发如果没有红区、契约 owner 和集成节奏，最终只会把冲突推迟到合并阶段集中爆炸。
3. 当前正式文档已经收敛成 Ring0 / Ring1 / Ring2 体系，需要一套并行开发流程继续服从这套真相源。

## 2. In Scope

1. 定义多 Agent / 多 Worktree 的正式工程规范。
2. 设计并行 Lane 切分、owned surface、红区文件和契约冻结策略。
3. 设计 Worktree 建立、分支命名、集成回归和 PR 门禁。
4. 将规则回填到 `AGENTS.md`、文档分层索引和工程规范文档。
5. 明确本轮由单一 Codex owner 完成启动物料，不在启动阶段伪装成多 Agent 已同时进入实现。

## 3. Out of Scope

1. 不直接实现自动创建 Worktree 的脚本。
2. 不替代 Linear 本身的任务管理能力。
3. 不在本主题内直接实施后续全部业务重构。
4. 不伪造 Linear Issue / Story 已建成；若本地无法直接建单，必须显式标记为外部 blocker。

## 4. Story Backlog

1. `PAR-S1`：并行 Lane 切分与 owned surface 矩阵固化
2. `PAR-S2`：多 Worktree 建立规范与本地环境隔离规则落地
3. `PAR-S3`：共享契约冻结与红区文件门禁建立
4. `PAR-S4`：集成值班、回归节奏与 PR 合并顺序固化
5. `PAR-S5`：Linear / Story / PR / Worktree 的一一映射规则固化

5. 所有无法在本地闭环的动作，必须显式记为待办，不得伪造完成状态。
