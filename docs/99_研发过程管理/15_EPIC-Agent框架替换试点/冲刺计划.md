# 冲刺计划

## 1. 目标

本轮冲刺目标是把 EPIC15 从替换方向讨论推进到可执行试点状态。

本轮完成口径：

1. 固化 Agent 状态机与 graph state 映射范围。
2. 确认 `FactoryAgent` façade 与新编排内核的单向切换方式。
3. 明确 `nanobot`、`PydanticAI`、`LangGraph`、`opencode` 的职责分工。
4. 形成新编排内核的合同回归测试入口与替换验收口径。

## 2. 当前状态

当前状态：已启动（待进入 Story 实施）

当前判断：

1. EPIC15 承接 EPIC4 之后的第一项替换试点。
2. 当前最适合替换的是 Agent 编排骨架，而不是 Runtime 语言与控制层。
3. `nanobot` 已明确保留，但仅保留为模型 / Provider 访问适配层。
4. 本主题采用单向迁移策略，不再保留旧编排链路切换机制。
5. 当前已完成 Epic 级文档与 Story 级文档准备，可正式进入 `EPIC15-S1` 开发。

## 4. 启动期执行节奏

### 4.1 第 0-24 小时

1. 固化 Agent 状态图和 graph state 映射边界。
2. 确认 façade 保留策略与单向切换方案。
3. 输出 `nanobot / LangGraph / PydanticAI / opencode` 的职责矩阵。

### 4.2 第 24-48 小时

1. 启动 façade 背后的新编排内核接线。
2. 先迁移 `DISCOVERY / ALIGN_IO / BLUEPRINT_LOOP`。
3. 建立新编排内核合同回归测试入口。

### 4.3 启动期附加规则

1. 不在本主题内提前改 Runtime 控制层语言。
2. 不在本主题内同时做 Agent 框架替换和内部服务契约替换。
3. 对外 façade 合同未被测试保护前，不得直接切换默认实现。
4. 若迁移影响正式 Agent 设计边界，必须同步回填 Ring1 文档。
5. 不再为旧编排链路保留长期切换入口。

## 5. 首轮验收口径

本轮冲刺至少满足以下条件：

1. Agent 状态图试点范围已固定。
2. graph state 与 `orchestration_context.v1` 的映射口径已形成。
3. façade 单向切换方案已明确。
4. 新编排内核合同回归测试入口已建立。
5. `nanobot` 的降级定位已成为正式过程口径。
