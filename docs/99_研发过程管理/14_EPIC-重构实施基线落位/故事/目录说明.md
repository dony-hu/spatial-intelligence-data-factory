# 故事目录说明

本主题用于承接“全面重构”的第一阶段实施 Story，重点解决共享契约、legacy 边界、主链结构拆分与测试基线问题。

当前 Story 已按以下顺序补齐：

1. `EPIC4-S1`：共享契约冻结与消费基线固化
2. `EPIC4-S2`：legacy 边界冻结与目录归属声明
3. `EPIC4-S3`：Factory Agent 巨石拆分但保持外部契约不变
4. `EPIC4-S4`：Runtime / API seam 收敛与主链边界清理
5. `EPIC4-S5`：旧测试口径清理与 baseline 建立
6. `EPIC4-S6`：集成回归与首轮并行开发准入

每个 Story 文件应显式声明：

1. 所属 Lane
2. owned surface
3. 影响的共享契约
4. 最小测试集
5. 是否影响 Ring1 正式文档

当前启动口径：

1. `EPIC4-S1` 与 `EPIC4-S2` 是首轮实施前置。
2. `EPIC4-S3` 与 `EPIC4-S4` 只有在 `S1 / S2` 收口后才进入真实代码改动。
3. `EPIC4-S5` 跟随 `S3 / S4` 同步推进，不允许末尾补 baseline。
4. `EPIC4-S6` 只负责准入与收口，不承接新的结构性迁移。
