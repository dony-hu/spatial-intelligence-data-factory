# 研发主题：重构实施基线落位

## 1. 主题目标

基于已完成的新设计文档、软件架构选型和多 Agent / 多 Worktree 并行开发规范，正式启动“全面重构”的第一阶段实施主题，解决以下问题：

1. 新设计文档已经明确，但代码仍处于新旧实现并存的过渡态。
2. 共享契约、目录归属、主链路 seam 和测试口径尚未完全收敛。
3. 如果直接进入多 Lane 全面改造，容易在契约、目录和回归三层同时失控。
4. 后续 Agent 框架替换、核心组件替换和编程语言替换，需要一个稳定的实施基线作为前置条件。

## 2. In Scope

1. 冻结并固化共享契约，包括：
   - `workpackage_schema.v1`
   - `orchestration_context.v1`
   - Agent 与 Runtime 交接载荷
2. 明确 `src/` legacy/compat 边界与目标 owned surface。
3. 启动 `packages/factory_agent/` 的结构拆分，但保持外部正式契约稳定。
4. 启动 `services/governance_api/` 与 `services/governance_worker/` 的 seam 收敛。
5. 建立本主题的 smoke baseline、known-fail 清单和最小验证入口。
6. 为后续并行开发输出准入条件和集成门禁。

## 3. Out of Scope

1. 不在本主题内正式切换到 `Go + Temporal` Runtime 主链。
2. 不在本主题内全量切换到 `LangGraph + PydanticAI` Agent 主框架。
3. 不在本主题内全量落地 `Buf + Protobuf + ConnectRPC`。
4. 不在本主题内进行前端大规模重构。
5. 不在本主题内重写治理算法实现。
6. 不把本主题误做成“最终目标架构全部落地”。

## 4. Story Backlog

1. `EPIC4-S1`：共享契约冻结与消费基线固化
2. `EPIC4-S2`：legacy 边界冻结与目录归属声明
3. `EPIC4-S3`：Factory Agent 巨石拆分但保持外部契约不变
4. `EPIC4-S4`：Runtime / API seam 收敛与主链边界清理
5. `EPIC4-S5`：旧测试口径清理与 baseline 建立
6. `EPIC4-S6`：集成回归与首轮并行开发准入

## 5. 完成定义（DoD）

满足以下全部条件视为本主题完成：

1. 共享契约已冻结，并有明确 owner 与唯一消费口径。
2. `src/` 已明确标注为 legacy/compat，新增能力默认进入目标 owned surface。
3. Factory Agent、Runtime / API 至少各完成一轮结构拆分，且外部正式契约保持稳定。
4. 本主题的 smoke baseline 可重复执行，known-fail 清单显式存在。
5. 后续并行 Lane 的准入条件、最小测试集和集成门禁已明确。
6. 本主题下的结论已能支撑后续 Agent 框架替换试点和 Runtime 控制层替换试点。
