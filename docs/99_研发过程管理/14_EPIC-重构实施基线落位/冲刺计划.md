# 冲刺计划

## 1. 目标

本轮冲刺目标不是完成最终重构，而是把 EPIC4 从主题定义推进到可执行实施状态。

本轮完成口径：

1. 形成首版共享契约冻结清单。
2. 明确 legacy/target 边界和首轮 owned surface。
3. 为 Factory Agent 与 Runtime / API 的结构拆分建立切入点。
4. 固化 smoke baseline、known-fail 清单和统一验证入口。
5. 为后续并行开发提供明确准入条件。

## 2. 当前状态

当前状态：已启动（待进入 Story 实施）

当前判断：

1. 仓库已具备目标架构文档和并行开发规范。
2. 代码仍处于新旧实现并存的过渡态。
3. EPIC4 应优先收敛实施基线，再进入后续替换试点。
4. 当前已补齐首轮 Story 文档与启动检查单，但尚未进入真实结构拆分提交。

## 4. 启动期执行节奏

### 4.1 第 0-24 小时

1. 输出共享契约冻结清单。
2. 固化 `src/` legacy/compat 声明。
3. 确认 EPIC4 的首轮 owned surface 和红区 owner。
4. 建立统一验证入口，默认使用 `.venv/bin/python -m pytest`。

### 4.2 第 24-48 小时

1. 启动 `S3` 和 `S4` 的最小结构拆分切片。
2. 同步建立 smoke baseline 与 known-fail 清单。
3. 明确哪些测试失败属于旧 seam，哪些属于现行主链问题。

### 4.3 启动期附加规则

1. 每日只进入一个共享契约类 PR。
2. 共享契约未合并前，下游 Lane 不得提前实现猜测版接口。
3. 单个 PR 不同时承接契约迁移、目录迁移和大规模实现迁移。
4. 若结构拆分影响正式边界，必须同步回填 Ring1 正式文档。

## 5. 首轮验收口径

本轮冲刺至少满足以下条件：

1. 共享契约冻结清单已经形成。
2. legacy/target 边界已在过程文档中明确。
3. Factory Agent 与 Runtime / API 已明确首轮结构拆分切口。
4. smoke baseline 与 known-fail 清单已建立。
5. EPIC4 后续 Story 可以进入并行实施。
