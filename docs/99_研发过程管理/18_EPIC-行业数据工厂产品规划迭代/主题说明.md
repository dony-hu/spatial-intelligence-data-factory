# 研发主题：行业数据工厂产品规划迭代

## 1. 主题目标

围绕当前“空间智能数据工厂”正式样板能力，开展一轮面向“行业数据工厂”的产品规划迭代，重点回答以下问题：

1. 哪些能力应被定义为跨行业稳定产品内核。
2. 哪些能力属于行业适配层，应通过 bundle、schema、可信数据接线和看板模板承接。
3. 首批行业试点应满足什么选择条件，而不是凭直觉直接锁定行业。
4. 哪些结论应继续停留在 Ring2 过程层，哪些结论稳定后需要回填到 Ring1 正式产品文档。

## 2. In Scope

1. 复用当前 Ring1 产品与架构真相源，明确“地址治理样板”与“行业数据工厂产品层”的关系。
2. 输出本主题 `产品说明.md`，形成产品定位、角色、场景、指标、版本路线的过程基线。
3. 把本轮规划拆解为可执行 Story backlog，并明确依赖与建议 owner。
4. 形成后续回填 `docs/01_产品与业务/` 的差异清单与门禁。

## 3. Out of Scope

1. 不在本主题内直接修改 Runtime、Agent、Worker 或前端实现代码。
2. 不在本主题内直接把当前正式产品口径从“空间智能数据工厂”改写为新的 Ring1 名称。
3. 不在本主题内一次性锁定所有行业模板、商业定价或销售打法。
4. 不伪造 Linear Epic / Story 已创建；本地无法建单时必须显式标记为外部 blocker。

## 4. Story Backlog

1. `EPIC18-S1`：行业数据工厂产品边界与行业选择标准收敛
2. `EPIC18-S2`：跨行业共性能力与行业适配层能力地图重构
3. `EPIC18-S3`：目标角色、核心场景与度量口径升级
4. `EPIC18-S4`：版本路线图、交付模式与首批试点准入条件冻结
5. `EPIC18-S5`：Ring1 回填差异清单、Linear 映射与启动门禁收口

## 5. 完成定义（DoD）

满足以下全部条件视为本主题完成：

1. 行业数据工厂的产品边界、行业选择标准和非目标已冻结。
2. “跨行业稳定内核 / 行业适配层 / 交付模板层”三层口径已拆开。
3. 目标角色、核心场景、关键指标和版本路线已形成统一过程基线。
4. 已明确需要回填的 Ring1 文档与回填顺序，不再把过程口径长期悬空。
5. Linear Epic / Story 建单要求已写清；若仍未建单，已标记为外部 blocker。
