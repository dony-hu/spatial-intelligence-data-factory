# 研发主题：E2E全链路测试工业化完善

## 1. 主题目标

基于 PM 评审结果，补强当前 E2E 全链路测试设计，使其从“目录级覆盖”提升到“可直接实施、可直接验收、可直接发布门禁”的工业级测试规格。

本主题要解决的核心问题是：

1. E2E 文档当前只有摘要级目录，缺少详细用例模板和实施字段。
2. Factory Agent 的构建阻塞、门禁拒绝和恢复路径没有形成完整 E2E 规格。
3. Runtime 的审批态、回滚态和人工介入态没有在 E2E 里形成完整闭环。
4. `blocked` 结果没有按责任域和原因码分型。
5. 血缘、回放和数据湖产物层缺少一条一次性打穿的完整 E2E 链路。

## 2. In Scope

1. 补齐正式 `docs/09_测试与验收/` 下的 E2E 详细用例模板。
2. 补强 `全链路测试设计.md` 的整改清单和详细设计要求。
3. 将评审 findings 拆分为可执行 Story，并接入研发过程管理。
4. 收敛 Agent / Runtime / 血缘 / 数据湖相关 E2E 断言口径。

## 3. Out of Scope

1. 不实现自动化测试代码。
2. 不修改 runtime、worker、Factory Agent 业务逻辑。
3. 不调整现有单元测试和集成测试数量配额。

## 4. Story Backlog

1. `TEST-E2E-S1`：E2E 详细用例模板与必填字段补齐
2. `TEST-E2E-S2`：Agent 构建阻塞与门禁拒绝链路 E2E 补齐
3. `TEST-E2E-S3`：Runtime 审批、回滚与人工介入链路 E2E 补齐
4. `TEST-E2E-S4`：blocked 归因分型与环境矩阵收敛
5. `TEST-E2E-S5`：血缘、回放与数据湖产物全链 E2E 补齐

## 5. 完成定义（DoD）

满足以下全部条件视为本主题完成：

1. 所有 E2E 用例在进入自动化实现前均可按统一模板补齐详细设计。
2. Agent 的 `WAIT_USER_INPUT`、`WAIT_USER_GATE`、`BLOCKED` 和恢复点有正式 E2E 用例覆盖。
3. Runtime 的 `APPROVAL_PENDING`、`CHANGESET_READY`、`NEEDS_HUMAN`、`ROLLED_BACK` 有正式 E2E 用例覆盖。
4. `blocked` 结果至少能区分外部 LLM、能力缺口、关键 key 缺失、基础设施不可用四类责任域。
5. 至少有一条 E2E 用例一次性验证 `source_snapshot_id -> input_binding_ref -> publish_id -> task_id -> trace_id -> evidence_ref -> canonical_record` 全链血缘。
