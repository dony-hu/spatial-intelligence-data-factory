# 研发主题 - BMAD 文档体系初始化

> 文档状态：当前有效
> 角色：BMAD 在本项目中的项目级过程管理入口
> 生效日期：2026-03-06

## 1. 目标

在不修改现有 `docs/` 真相源文件的前提下，为 BMAD 建立一套可落地的文档初始化与映射规则，使 BMAD 能基于当前数据工厂文档体系推进：

1. 项目级目标澄清与 PRD 收敛。
2. 架构主线收敛与主题拆分。
3. 研发主题、故事、冲刺、评审、复盘的持续管理。
4. 与 SpecKit 的模块级规格产物形成可追踪交接。

## 2. 单一事实源

BMAD 在本项目中不得另起一套“正式文档主线”，必须服从以下入口：

1. 文档总入口：`docs/00_阅读指南.md`
2. 架构真相源入口：`docs/02_总体架构/架构索引.md`
3. 项目状态入口：`docs/99_研发过程管理/工作流状态.yaml`
4. 研发主题入口：`docs/99_研发过程管理/`
5. 仓库协作约束：`AGENTS.md`

`_bmad-output/` 仅作为过程产物和证据区，不替代正式真相源。

## 3. BMAD 在本项目中的定位

BMAD 负责“项目级与主题级”文档编排，重点覆盖：

1. 产品简述、PRD、架构主线。
2. Epic/主题拆分与 Story 管理。
3. 冲刺计划、评审记录、复盘记录。
4. 与测试、验收、证据索引的项目级收口。

BMAD 不负责替代模块级 `spec.md / plan.md / tasks.md / data-model.md / contracts/` 的细化工作；这些内容由 SpecKit 主导。

## 4. 与当前文档体系的映射边界

1. 正式产品与架构文档继续沉淀在 `docs/01_产品与业务/`、`docs/02_总体架构/`、`docs/03_数据处理工艺/`、`docs/04_系统组件设计/`、`docs/07_系统运行与运维/`、`docs/08_AI能力设计/`、`docs/10_研发与工程规范/`、`docs/11_附录/`、`docs/09_测试与验收/`。
2. BMAD 的项目级流程文件与主题级过程文档统一挂在 `docs/99_研发过程管理/`。
3. `docs/99_研发过程管理/02_BMAD/` 只保存 BMAD 使用说明、映射规则和初始化约定，不承接具体业务主题内容。

## 5. 对“规划中文档”的处理方式

阅读指南中尚未创建但已规划的文档，按以下方式进入 BMAD 主线：

1. BMAD 直接主导：
- `docs/03_数据处理工艺/数据处理总流程.md`
- `docs/07_系统运行与运维/系统部署架构.md`
- `docs/10_研发与工程规范/代码规范.md`
- `docs/10_研发与工程规范/版本管理策略.md`
- `docs/11_附录/常见问题.md`

2. BMAD 定题、SpecKit 落细：
- `docs/03_数据处理工艺/地址标准化工艺.md`
- `docs/03_数据处理工艺/地址真实性核验工艺.md`
- `docs/03_数据处理工艺/空间实体治理工艺.md`
- `docs/03_数据处理工艺/多源数据融合工艺.md`
- `docs/08_AI能力设计/检索与召回机制.md`
- `docs/08_AI能力设计/推理与排序策略.md`
- `docs/08_AI能力设计/模型评估机制.md`

## 6. 与 SpecKit 的交接原则

1. BMAD 先定义主题、边界、验收口径和 Story。
2. 当某个 Story 需要模块级接口、数据模型、契约和任务拆解时，转交 SpecKit。
3. SpecKit 产物稳定后，由 BMAD 把其中的长期真相回填到正式 `docs/` 或研发主题文档。

## 7. 当前初始化输出

本主题初始化包含：

1. `主题说明.md`：说明 BMAD 在本项目中的定位。
2. `设计说明.md`：说明 BMAD 与正式文档、SpecKit 的协作规则。
3. `故事/目录说明.md`：定义 BMAD 主题下 Story 的落点约束。
4. `文档映射配置.yaml`：提供机器可读的 BMAD 文档映射配置。
