# BMAD 故事目录说明

本目录用于保存 BMAD 主题下的故事级正式文档。

## 1. 命名约定

1. 使用 `<主题缩写>-<序号>-<中文标题>.md`
2. 示例：
- `MVP-A1-CLI-Agent-LLM-对话确认治理需求.md`
- `OBS-PG-S1-系统可观测性与PG库一体化构建.md`

## 2. 内容约定

每个故事文件至少包含：

1. 背景与目标
2. In Scope / Out of Scope
3. 验收标准
4. 测试与证据要求
5. 关联代码与文档路径
6. 是否需要下钻到 SpecKit feature

## 3. 与 SpecKit 的关系

当故事需要模块级细化时，在故事文件中补充以下字段：

1. `SpecKit Feature ID`
2. `SpecKit 目录`
3. `回填目标文档`

Story 文件仍是主题级过程真相源，不能被 `specs/<feature-id>/tasks.md` 替代。
