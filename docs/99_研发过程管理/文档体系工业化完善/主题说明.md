# 研发主题：文档体系工业化完善

## 1. 主题目标

基于 A-TW 架构与技术写作联合评审结果，完成一轮面向工业级软件交付标准的正式文档体系收敛，解决以下问题：

1. 正式文档缺口仍然存在，无法单独支撑部署、运维、AI 评估和工程协作。
2. 正式专题文档与过程管理文档之间存在现行口径冲突。
3. Factory Agent 与 Runtime 之间缺少显式交接契约。
4. 数据库域设计已经收敛，但 DDL 所有权、跨界约束和观测口径仍需补齐。
5. 关键正式文档的图注与说明仍有模板化痕迹，可读性未达到工业级设计文档要求。

## 2. In Scope

1. 补齐 `docs/` 正式阅读体系中缺失的长期有效文档。
2. 新增 Agent 与 Runtime 交接契约文档并接入现有索引。
3. 收敛数据库、观测、地址治理工艺等正式专题中的旧口径。
4. 修正会反向污染实现的过程管理文档与冲刺文档。
5. 提升关键正式文档图注质量和浏览体验。

## 3. Out of Scope

1. 不修改 runtime、worker、业务实现代码。
2. 不重写整个 BMAD/SpecKit 方法资产。
3. 不清理历史归档区的所有旧文档，只处理当前仍会误导实现的内容。

## 4. Story Backlog

1. `DOC-S1`：正式文档缺口补齐
2. `DOC-S2`：Agent 与 Runtime 交接契约补齐
3. `DOC-S3`：数据库与可观测设计口径收敛
4. `DOC-S4`：过程管理文档现行口径对齐
5. `DOC-S5`：关键文档图注与工业化可读性收敛

## 5. 完成定义（DoD）

满足以下全部条件视为本主题完成：

1. `docs/00_阅读指南.md` 中本轮涉及的规划文档已经落地为正式文件。
2. Agent / Runtime / 数据库 / 可观测设计之间不存在显式冲突口径。
3. 过程管理区中仍被直接消费的文档不再推动旧状态机、旧表名或旧数据域口径。
4. 新增文档全部接入正式索引，链接有效。
5. 所有新增 Mermaid 流程图采用纵排，并配中文图说明。
