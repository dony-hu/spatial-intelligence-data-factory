# 设计说明

## 1. 设计原则

本主题遵循五条原则：

1. 正式文档优先补齐长期有效知识，不把阶段性材料继续堆到阅读主线。
2. 同一项正式知识只能保留一个现行口径，过程文档如有差异必须显式标注。
3. 工业级文档必须覆盖部署、运维、评估、边界、变更治理和恢复机制。
4. 图的职责是帮助决策，不是装饰，因此优先拆图而不是硬塞一张大图。
5. 所有整改都保持 docs-only，不越过运行时代码边界。

### 2.3 口径收敛

重点收敛：

1. `trust_db -> trust_data` 的正式/过渡关系
2. `addr_*` 旧表名与现行 `governance/runtime/audit` 口径
3. 旧状态机 `PENDING/RUNNING/...` 与现行 Agent/Runtime 状态机

### 2.4 过程管理区治理

对仍可能误导实现的过程文档，采取两种方式：

1. 现行有效：直接改成当前口径
2. 历史阶段：在文首显式标出历史属性，并给出现行跳转
