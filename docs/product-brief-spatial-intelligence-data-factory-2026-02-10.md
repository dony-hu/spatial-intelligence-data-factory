# Product Brief - 空间智能数据工厂

## 1. 背景与问题

当前组织已具备数据湖/数仓、基础 ETL 与初步治理能力，但仍存在以下问题：

- 数据采集、清洗、计算逻辑复用不足，重复开发成本高。
- 数据质量缺乏统一自动化检测，问题发现滞后。
- 数据流程可追踪性不足，定位与审计效率低。
- 批流任务协同能力不足，难以稳定支撑准实时业务。
- 业务闭环（实体发现、核验、人工复核、现场核实）链路断点较多。

## 2. 产品愿景

建设一个可复用、可治理、可监控、可扩展的数据工厂平台，形成标准化的数据生产与消费能力，支持空间智能场景持续交付。
在业务场景不变的前提下，引入 Agent 驱动执行内核，强化从需求到交付的自动化与人机协同能力。

## 3. 业务目标（阶段性）

1. 标准化数据生产流程：统一采集、清洗、转换模型与任务模板。
2. 建立数据质量闭环：规则校验、统计校验、异常检测与告警。
3. 建立全流程可追踪体系：血缘、执行日志、运行快照、变更审计。
4. 提升交付效率与稳定性：组件复用、规范执行、自动化回滚与恢复。
5. 支撑批流一体与实时消费：统一调度、监控与 API 化交付。
6. 建立 Agent 自治执行体系：实现目标驱动、自主决策、工具调度、状态记忆的闭环执行。

## 4. 范围定义

### 4.1 In Scope

- 多子项目共性能力沉淀与统一治理（上海、昆山、吴江、北京、深圳宝安）。
- 数据任务标准化模板与执行规范。
- 数据质量规则引擎与质量看板。
- 数据血缘与运行日志统一采集与查询。
- 实体发现与运营闭环编排（自动核验 + 人工确认 + 现场任务）。
- 面向 BI/模型/报告的稳定数据 API 交付能力。
- 城市运行指挥调度能力基座（语音驱动、场景化摄像头调度接口与策略）。
- Agent 控制平面能力（目标编排、决策循环、工具调用、状态记忆、执行留痕）。

### 4.2 Out of Scope（当前阶段）

- 通用低代码平台化能力（面向全公司所有业务线的一次性覆盖）。
- 全量历史数据一次性重建（仅支持按优先级分批迁移）。
- 非核心场景的自定义算法市场与插件生态。
- 各地区项目特有的定制字段、定制流程、临时交付需求（保留在子项目层）。

## 5. 关键用户与使用场景

- 数据工程师：快速构建与复用任务链路，降低开发与维护成本。
- 数据治理/质检人员：定义与执行质量规则，跟踪异常闭环。
- 业务分析与应用团队：稳定获取可追踪、可解释的数据资产。
- 运营与审核团队：处理实体异常、人工确认与现场核实任务。

核心场景：
- 地址治理与结构化处理。
- 地址/空间实体关联与多源融合。
- 自动预标注 + 人工标注 + 抽检复核闭环。
- 文档（PDF/Word/BIM）实体抽取与属性融合。
- 未识别实体发现、核验与运营闭环执行。
- 城市运行指挥大厅场景下的语音驱动摄像头精准调度。

## 6. 成功指标（KPI）

- 数据任务开发周期提升 >= 50%
- 数据质量自动检测覆盖率 >= 90%
- 数据任务失败自动恢复率 >= 95%
- 数据血缘完整率 >= 98%
- 实时链路延迟 <= 30s
- 未识别实体自动发现召回率 >= 90%
- 自动核验后无需人工比例 >= 60%（阶段目标）
- 平台能力复用率（北极星指标）：被两个及以上地区复用的能力占比持续提升

## 7. 约束与风险

### 7.1 约束

- 需要兼容现有数据湖/数仓与 ETL 体系。
- 多角色协作下必须保证流程可审计与责任可追踪。
- 需满足项目规范中的日志、测试数据与交付要求。
- 关键自动化决策需保留人工审批门禁（指标口径、合规、发布、SLA）。

### 7.2 风险

- 源数据质量波动导致规则误报/漏报。
- 批流链路耦合不当导致稳定性下降。
- 实体运营闭环跨团队协作成本高，影响落地速度。

## 8. 里程碑（建议）

1. M1（0-2 个月）：基础框架与最小可运行链路上线。
2. M2（2-4 个月）：核心模块标准化，质量规则可执行。
3. M3（4-6 个月）：可视化编排与元数据查询能力上线。
4. M4（6-9 个月）：全链路 SLA、运维 SOP 与治理手册完善。

## 9. 验收标准（进入 PRD 前）

- 范围边界（In/Out Scope）已评审通过。
- KPI 与阶段目标已确认且可量化。
- 核心用户画像与关键场景已冻结。
- 关键风险与约束已形成应对策略。
- 可以据此拆解 PRD 模块与需求优先级。

## 10. 已确认决策（2026-02-10）

1. 本期优先级最高的 2 个业务场景：
   - 公安地址治理能力基座（覆盖上海、吴江、北京等场景共性能力）。
   - 城市运行调度能力基座（以深圳宝安指挥调度场景为牵引）。
2. 首期上线范围：多区域并行（分批上线、统一能力底座）。
3. 北极星指标：平台能力复用率。
