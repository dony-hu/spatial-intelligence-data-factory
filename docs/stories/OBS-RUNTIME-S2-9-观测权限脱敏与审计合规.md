# Story：OBS-RUNTIME-S2-9 观测权限、脱敏与审计合规

## 1. 目标

为运行态可观测下钻能力补齐**单租户最小 RBAC**与数据脱敏，满足最小合规要求。

## 2. 范围

1. 角色分级访问控制：
   - 运营只读
   - 值班可 ACK 告警
   - 管理员可导出与回放
   - 单租户范围内生效，不引入多租户维度
2. 字段级脱敏策略：
   - 地址明细脱敏
   - evidence 中敏感字段脱敏
3. 敏感操作审计：
   - 访问任务详情
   - 导出数据
   - 告警 ACK 与升级
4. 合规审计查询接口（只读）。

## 3. 验收标准

1. 未授权用户无法访问敏感下钻信息。
2. 脱敏规则对页面与 API 同步生效。
3. 所有敏感操作有审计事件可回查。
4. 导出能力遵守角色限制。
5. 任何鉴权失败返回显式权限错误。
6. 不引入多租户 RBAC 设计与实现（明确排除）。

## 4. 测试要求（TDD）

1. 先补失败用例：
   - RBAC 鉴权测试
   - 脱敏一致性测试
   - 审计事件完整性测试
2. 再实现权限与脱敏逻辑。
3. 最后执行合规回归测试。

## 5. 依赖

1. S2-4 下钻与回放能力。
2. 审计事件模型包含 actor 与 action 字段。
