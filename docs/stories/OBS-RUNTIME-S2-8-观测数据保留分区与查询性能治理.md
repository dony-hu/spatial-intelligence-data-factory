# Story：OBS-RUNTIME-S2-8 观测数据保留分区与查询性能治理

## 1. 目标

建立观测数据容量治理能力，保证可观测页面在数据增长后仍保持稳定响应。

## 2. 范围

1. `audit.events` 与高频观测表的时间分区策略。
2. 数据保留与归档策略（热/冷数据分层）。
3. 聚合查询索引优化与慢 SQL 治理。
4. 回放查询性能基线与压测脚本。

## 3. 验收标准

1. 查询性能满足目标：
   - 首屏聚合 API `<1.5s`
   - 任务下钻 `<800ms`
2. 分区与归档策略可执行并可回查。
3. 慢 SQL 告警可识别高风险查询。
4. 历史数据增长下性能无明显退化。
5. 变更全部通过 Alembic，不引入旁路 DDL。

## 4. 测试要求（TDD）

1. 先补失败用例：
   - 性能门槛测试
   - 分区查询正确性测试
   - 归档回查测试
2. 再实现索引/分区/归档逻辑。
3. 最后执行压测与回归。

## 5. 依赖

1. 当前 PG schema 已稳定。
2. 观测 API 查询路径固定。
