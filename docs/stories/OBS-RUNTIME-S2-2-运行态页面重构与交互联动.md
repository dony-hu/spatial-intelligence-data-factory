# Story：OBS-RUNTIME-S2-2 运行态页面重构与交互联动

## 1. 目标

将当前可观测页面重构为“系统运行态可观测总览”，突出治理结果与风险定位。

## 2. 范围

1. 页面信息架构重排：`KPI -> 风险趋势 -> 版本对比 -> 明细下钻`
2. 顶部筛选：window/ruleset/status/version
3. 图表和列表联动（点击图表自动过滤任务列表）
4. 移除或弱化研发过程卡片

## 3. 验收标准

1. 默认显示最近 24h 运行态数据。
2. KPI 卡片 6 项完整可见。
3. 风险区展示阻塞 Top5 和低置信模式 Top5。
4. 筛选条件变化后，图表与明细同步更新。
5. 无数据时显示明确空态引导。

## 4. 测试要求（TDD）

1. 先补失败用例：
   - 页面结构测试
   - 筛选联动测试
   - 空态渲染测试
2. 再改页面实现。
3. 最后执行 UI 回归。

## 5. 依赖

1. S2-1 聚合 API 已可用。
2. 前端静态资源与模板加载链路稳定。
