# Story：OBS-RUNTIME-S2-7 地址治理数据质量漂移与异常检测

## 1. 目标

建立治理质量漂移检测能力，提前发现模型/规则/数据源变化导致的质量退化。

## 2. 范围

1. 新增质量指标：
   - 地址标准化覆盖率
   - 行政区匹配一致率
   - 低置信占比
   - 阻塞原因分布稳定性
2. 新增漂移检测：
   - 与 7d 基线对比
   - 与版本 baseline 对比
3. 异常触发与告警联动（质量退化 P1/P2）。
4. 页面新增“质量漂移”区块（趋势 + 异常点标记）。

## 3. 验收标准

1. 质量指标可按时间窗稳定查询。
2. 漂移检测可识别突增/突降异常。
3. 异常点可跳转到任务明细样本。
4. 版本对比可给出质量提升/退化结论。
5. 质量检测失败显式报错，不静默忽略。

## 4. 测试要求（TDD）

1. 先补失败用例：
   - 质量指标计算测试
   - 漂移阈值检测测试
   - 异常下钻联动测试
2. 再实现检测器与 API。
3. 最后回归并产出质量报告样例。

## 5. 依赖

1. S2-1 聚合 API。
2. S2-4 下钻能力。
