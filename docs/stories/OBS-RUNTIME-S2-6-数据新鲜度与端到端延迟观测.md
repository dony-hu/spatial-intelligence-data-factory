# Story：OBS-RUNTIME-S2-6 数据新鲜度与端到端延迟观测

## 1. 目标

建立从任务提交到页面展示的端到端时延可观测能力，定位“页面有数据但不及时”的问题。

## 2. 范围

1. 新增核心延迟指标：
   - `event_lag_seconds`
   - `aggregation_lag_seconds`
   - `dashboard_data_age_seconds`
2. 支持按 ruleset/version/status 维度切片查看延迟。
3. 延迟阈值触发告警并支持追踪根因（采集慢/聚合慢/查询慢）。
4. 页面新增“数据新鲜度”卡片与趋势图。

## 3. 验收标准

1. 指标在 24h 窗口可见且可筛选。
2. 任一延迟超阈值可触发告警。
3. 可定位延迟发生层级（ingestion/aggregation/query）。
4. 页面刷新后可显示最近一次数据更新时间。
5. 数据源不可用时显式报错，不返回虚假新鲜数据。

## 4. 测试要求（TDD）

1. 先补失败用例：
   - 延迟指标计算测试
   - 阈值告警测试
   - 页面新鲜度渲染测试
2. 再实现指标链路与页面展示。
3. 最后执行集成回归。

## 5. 依赖

1. S2-5 告警框架可用。
2. 审计事件包含关键时间戳字段。
