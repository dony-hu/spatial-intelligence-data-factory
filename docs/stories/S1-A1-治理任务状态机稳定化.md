# Story: S1-A1 治理任务状态机稳定化

## 目标

保证治理任务状态在主链路中稳定按 `PENDING -> RUNNING -> SUCCEEDED/FAILED` 演进，避免异常分支出现状态错乱。

## 验收标准

1. 任务提交后初始状态为 `PENDING`。
2. 执行开始后状态更新为 `RUNNING`。
3. 正常执行完成后状态为 `SUCCEEDED`，失败时状态为 `FAILED`。
4. 任一异常路径不允许返回空状态或未定义状态。

## 开发任务

1. 先补测试：增加状态流转失败用例（提交失败、执行异常、队列降级）。
2. 再改实现：校正 `tasks` 路由与 `governance_job` 状态写入时机。
3. 最后验证：运行目标测试集并输出状态流转证据。

## 测试用例

1. 正常提交+执行：最终 `SUCCEEDED`。
2. 模拟 worker 异常：最终 `FAILED`。
3. 队列不可用降级：状态仍可闭环到终态。
